## Day 18: Date Object Deep Dive 🕰️

JavaScript gives you a built-in `Date` object to work with dates and times. You can use it to get the current date, time, day, month, year, and even calculate differences between dates. This is super useful for features like:

- Displaying today's date on a page
- Showing timestamps for blog posts or messages
- Creating countdown timers
- Calculating age or duration

Let’s start with the basics in a way that actually feels useful.

<Divider />

## 🗓️ Creating a Date Object

There are a few ways to create a date, but the most common one is just:

<CodeBlock language="js">
{`let now = new Date();
console.log(now); // prints the current date and time`}
</CodeBlock>

This creates a `Date` object with the **current moment** in time.

<Note>
The output might look like: `Mon Apr 07 2025 10:15:30 GMT+0530 (India Standard Time)`
</Note>

<Divider />

## 📅 Get Individual Parts of the Date

Once you have a `Date` object, you can extract different parts using methods:

<CodeBlock language="js">
{`let now = new Date();

console.log(now.getFullYear()); // 2025
console.log(now.getMonth());    // 3 (April — zero-based!)
console.log(now.getDate());     // 7
console.log(now.getDay());      // 1 (Monday — 0 is Sunday)
console.log(now.getHours());    // 10
console.log(now.getMinutes());  // 15
console.log(now.getSeconds());  // 30`}
</CodeBlock>

<Tip>
Remember: `.getMonth()` returns values from `0 (Jan)` to `11 (Dec)`. Always add +1 if you want the human-readable month.
</Tip>

<Divider />

## 💡 Formatting a Date (Simple Way)

If you want to show the date in a readable format like `07/04/2025`, you can combine values:

<CodeBlock language="js">
{`let now = new Date();

let day = now.getDate();
let month = now.getMonth() + 1;
let year = now.getFullYear();

let formatted = day + "/" + month + "/" + year;
console.log(formatted); // e.g. "7/4/2025"`}
</CodeBlock>

We’ll keep it basic here and skip complex international formatting for now.

<Divider />

## ⏳ Get Timestamp (Milliseconds Since 1970)

Every date has a number behind it — the number of milliseconds since Jan 1, 1970.

<CodeBlock language="js">
{`let now = new Date();
console.log(now.getTime()); // something like 1743926730000`}
</CodeBlock>

This number is great when you want to **compare two dates** or do **countdowns**.

<Note>
This is called a "Unix Timestamp" or "Epoch time".
</Note>

## 📆 Date Comparisons

Working with dates often involves **comparing them** — whether you're checking if a deadline has passed or counting down to an event.

JavaScript allows you to compare `Date` objects directly, but the most **accurate** and **reliable** method is using `.getTime()` — which gives the number of milliseconds since January 1, 1970.

### Comparing Two Dates

<CodeBlock language="js">
{`const date1 = new Date("2024-01-01");
const date2 = new Date("2025-01-01");

console.log(date1.getTime() < date2.getTime()); // true
console.log(date1 < date2); // also true
console.log(date1.getTime() === date2.getTime()); // false`}
</CodeBlock>

<Note>
Using `.getTime()` is safer when comparing dates because it avoids unexpected behavior and handles edge cases better — especially when working with timezones.
</Note>

<Divider />

## 🕒 Countdown to a Future Date

Let’s build a small script that calculates **how many days are left until a future date**. This is great for countdowns like New Year or a product launch.

<CodeBlock language="js">
{`const today = new Date();
const targetDate = new Date("2025-01-01");

// Get time difference in milliseconds
const diff = targetDate.getTime() - today.getTime();

// Convert to days
const daysLeft = Math.ceil(diff / (1000 * 60 * 60 * 24));

console.log(\`Days left: \${daysLeft}\`);`}
</CodeBlock>

<Tip>
If the `daysLeft` is negative, it means the date is already in the past!
</Tip>
